#

issue polling

polls GitHub, Asana, or Linear for new issues. label an issue queued and the agent picks it up.

[ ]

containerized sessions

every session runs inside a Docker container. the container is the sandbox — Claude has full permissions.

//

concurrent workers

process multiple issues in parallel. each gets its own branch, container, and Claude session.

<>

PR lifecycle

creates PRs, addresses review comments, waits for CI, and auto-merges. full lifecycle management.

~

workflow engine

configurable state machine via .plural/workflow.yaml. customize the pipeline per-repository.

*

child sessions

Claude can delegate subtasks to child sessions via MCP tools. the supervisor waits for all children before creating a PR.

configurable workflow

define your pipeline as a step function

drop a .plural/workflow.yaml in your repo. define states, actions, events, and transitions — or let the defaults handle it.

.plural/workflow.yaml 
source:
  provider: github            # github | asana | linear
  filter:
    label: queued

states:
  coding:
    type: task
    action: ai.code
    params:
      max_turns: 50
      max_duration: 30m
      system_prompt: file:.plural/prompt.md
    after:
      - run: "make lint"
    next: open_pr
    error: failed

  await_review:
    type: wait
    event: pr.reviewed
    params:
      auto_address: true
      max_feedback_rounds: 3

  merge:
    type: task
    action: github.merge
    params:
      method: squash            # rebase | squash | merge

custom prompts

inject system prompts inline or via file: paths. separate prompts for the coding phase and review phase.

after-hooks

run shell commands after any state. hooks receive $PLURAL_BRANCH, $PLURAL_PR_URL, and more via env vars.

CI failure policy

choose retry, abandon, or notify when CI fails. configure timeout per wait state.

add or reorder states

insert a github.comment_issue step, remove auto-merge, or rearrange the graph entirely.
