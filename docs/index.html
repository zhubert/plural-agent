label an issue. ship a PR.

autonomous headless daemon for Claude Code. polls for issues, writes code in containerized sessions, opens PRs, addresses review comments, and merges — no human in the loop.

install view on github

fully autonomous issue-to-merge pipeline

everything needed to run Claude Code headless

#

issue polling

polls GitHub, Asana, or Linear for new issues. label an issue queued and the agent picks it up.

[ ]

containerized sessions

every session runs inside a Docker container. the container is the sandbox — Claude has full permissions.

//

concurrent workers

process multiple issues in parallel. each gets its own branch, container, and Claude session.

<>

PR lifecycle

creates PRs, addresses review comments, waits for CI, and auto-merges. full lifecycle management.

~

workflow engine

configurable state machine via .plural/workflow.yaml. customize the pipeline per-repository.

*

child sessions

Claude can delegate subtasks to child sessions via MCP tools. the supervisor waits for all children before creating a PR.

how it works

1

label issue

add queued to a GitHub issue.

2

agent picks up

daemon polls and starts a containerized session.

3

claude codes

autonomous coding on a fresh branch.

4

PR opened

PR created with closing reference to the issue.

5

review + merge

addresses feedback, waits for CI, merges.

configurable workflow

define your pipeline as a state machine

drop a .plural/workflow.yaml in your repo to customize the agent's behavior. define states, actions, events, and transitions. the agent uses sensible defaults when no config is present.

$ plural-agent agent --repo owner/repo
